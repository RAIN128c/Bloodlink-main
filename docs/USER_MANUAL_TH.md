เอกสารนี้รวบรวมวิธีการใช้งานระบบสำหรับผู้ใช้งานแต่ละระดับ เพื่อให้เกิดประสิทธิภาพสูงสุดในการคล่องตัวทางการแพทย์และข้อผิดพลาดเป็นศูนย์ (Zero-Error Workflow)

---

## 1. การเข้าสู่ระบบ และการตั้งค่า PIN

### 1.1 เข้าสู่ระบบ (Login)
1.  เปิดเว็บเบราว์เซอร์ แล้วไปที่ URL ของระบบ
2.  กรอก **Email** และ **Password** เพื่อล็อกอิน
3.  ระบบจะพาไปยังหน้า Dashboard โดยอัตโนมัติ

### 1.2 ลืมรหัสผ่าน
1.  คลิก **"ลืมรหัสผ่าน"** ที่หน้า Login
2.  กรอก **อีเมล** ที่ใช้ลงทะเบียน
3.  กดปุ่ม **"ส่งลิงก์รีเซ็ต"** (ลิงก์มีอายุ 15 นาที)
4.  เปิดอีเมล แล้วคลิกลิงก์เพื่อตั้งรหัสผ่านใหม่

### 1.3 การตั้งค่า PIN (ลายเซ็นอิเล็กทรอนิกส์)
*   เมื่อเข้าใช้งานครั้งแรก ให้ไปที่เมนู **"โปรไฟล์ (Profile)"** เพื่อตั้งค่ารหัส PIN 6 หลัก
*   PIN นี้จะใช้สำหรับยืนยันการทำรายการสำคัญ เช่น การลงนามผลแล็บ และการอนุมัติใบส่งตรวจ

### 1.4 ส่วนประกอบของหน้าจอ

| ส่วน | คำอธิบาย |
|------|----------|
| **เมนูด้านซ้าย (Sidebar)** | นำทางไปยังหน้าต่างๆ (Dashboard, นัดหมาย, ผลตรวจ ฯลฯ) |
| **แถบด้านบน (Header)** | ข้อมูลผู้ใช้, แจ้งเตือน, ออกจากระบบ |
| **ป้ายตัวเลขสีแดง** | แสดงจำนวนรายการใหม่ที่ยังไม่ได้ตรวจสอบ (ข้อความ/คิว) |

## 2. หน้า Dashboard

หน้าหลักแสดงภาพรวมของระบบ โหลดข้อมูลทันทีจากเซิร์ฟเวอร์ (Server Component)

*   **ทางลัด (Shortcuts):** ปุ่มลัดไปยังนัดหมาย, สถานะผลตรวจ, และจัดการระบบ อยู่ด้านบนของหน้าจอ
*   **คิวงานล่าสุดวันนี้ (Today's Live Queue):** แสดง 5 ผู้ป่วยล่าสุดที่มีการอัปเดตสถานะวันนี้
*   **นัดหมายวันนี้ (Today's Appointments):** แสดงรายชื่อผู้ป่วยที่มีนัดหมายวันนี้
*   **สรุปจำนวน:** กราฟและตัวเลขจำนวนผู้ป่วยแยกตามสถานะ
*   **ค้นหา:** พิมพ์ HN หรือชื่อผู้ป่วยเพื่อค้นหาอย่างรวดเร็ว

## 3. สำหรับเจ้าหน้าที่พยาบาล และแพทย์ (Clinical Staff)

### 3.1 การลงทะเบียนผู้ป่วยใหม่
1.  ไปที่เมนู **"ผู้ป่วย" (Patients)** → กดปุ่ม **"+ ลงทะเบียนผู้ป่วย"**
2.  กรอกข้อมูล: เลข HN, ชื่อ-นามสกุล, เลขบัตรประชาชน (13 หลัก), เบอร์โทรศัพท์, ข้อมูลญาติ
3.  **ติ๊กเลือกโรคประจำตัว** (สามารถเลือกได้มากกว่า 1 โรค)
4.  กดบันทึก

### 3.2 การบันทึกข้อมูลก่อนส่งตรวจ (Pre-Lab)
1.  ค้นหาผู้ป่วยจากหน้าแดชบอร์ด แล้วกดเข้าสู่ "ประวัติผู้ป่วย"
2.  เลือกแท็บ **"บันทึกก่อนตรวจ (Pre-Lab)"**
3.  กรอก: ความดัน (BP1, BP2), ส่วนสูง, น้ำหนัก, รอบเอว, ชีพจร, อุณหภูมิ, DTX
4.  กดเลือกรายการตรวจเลือด (Multi-select: FBS, Lipid, CBC, HbA1c ฯลฯ)
5.  กดยืนยัน → ระบบจะสร้างคิวใบส่งตรวจไปยังฝั่งห้องปฏิบัติการทันที

### 3.3 การติดตามสถานะคิว
*   ไปที่เมนู **"สถานะผลตรวจเลือด" (Test Status)** เพื่อดูว่าผู้ป่วยอยู่ในขั้นตอนใด
*   สามารถสั่งพิมพ์ **"ใบส่งตรวจ (A5)"** ย้อนหลังได้ตลอดเวลา

### 3.4 สถานะผู้ป่วย (Workflow)

```
รอตรวจ → นัดหมาย → รอเจาะ → รอจัดส่ง → กำลังจัดส่ง → กำลังตรวจ → เสร็จสิ้น → รายงานผล
```

| สถานะ | คำอธิบาย |
|-------|----------|
| รอตรวจ | เพิ่งลงทะเบียนในระบบ |
| นัดหมาย | มีวันนัดตรวจแล้ว |
| รอเจาะ | รอเข้าห้องเจาะเลือด |
| รอจัดส่ง | เจาะเลือดเสร็จ รอจัดส่งตัวอย่าง |
| กำลังจัดส่ง | ส่งตัวอย่างเลือดไปห้องแล็บ |
| กำลังตรวจ | แล็บกำลังตรวจวิเคราะห์ |
| เสร็จสิ้น | ผลตรวจพร้อมแล้ว |
| รายงานผล | พิมพ์ผลตรวจและลงนามแล้ว |

## 4. สำหรับเจ้าหน้าที่ห้องปฏิบัติการ (MedTech)

### 4.1 การตรวจสอบคิว และสั่งพิมพ์แบบ Batch
1.  ไปที่เมนู **"คิวงานแล็บ" (Lab Queue)** ระบบจะแสดงรายชื่อผู้ป่วยที่รอตรวจ
2.  ติ๊กเลือกผู้ป่วยหลายคน → กดปุ่มเครื่องพิมพ์ → พิมพ์ **"ใบปะหน้า A4"** + **"ใบส่งตรวจ A5"** พร้อมกันทั้งปึก
3.  กด **"รับคิว"** (ต้องใส่ PIN) → คิวจะย้ายเข้า **"งานของฉัน (My Tasks)"** ป้องกันทำงานซ้ำซ้อน

### 4.2 การอัปโหลดผลแล็บ (Results Import & OCR)
1.  กรอกผลเลือดด้วยมือ หรือใช้ **OCR**: กดปุ่ม "อัปโหลดใบผลตรวจ" → ถ่ายรูป/เลือกไฟล์
2.  ระบบจะสกัดตัวเลขจากภาพมาใส่ในช่องฟอร์มให้อัตโนมัติ
3.  ตรวจสอบและแก้ไขค่าทศนิยมก่อนยืนยัน
4.  รองรับ **Bulk Upload** สำหรับนำเข้าผลแล็บหลายคนพร้อมกัน

### 4.3 การอนุมัติผล (E-Signature)
1.  ไปที่เมนู **"ผลตรวจเลือด" (Results)** → เลือกผู้ป่วย → กด **"พิมพ์เพื่ออนุมัติ"**
2.  ระบบจะบังคับใส่ **รหัส PIN 6 หลัก** ก่อนยืนยัน
3.  ระบบสร้าง Digital Signature พร้อม QR Code ตีตราประทับบนเอกสาร
4.  การ์ดผู้ป่วยจะหายจากกระดานแล็บ พร้อมแจ้งเตือนแพทย์อัตโนมัติ

## 5. ปฏิทินนัดหมาย (Appointments)

*   **มุมมอง:** เปลี่ยนเป็น รายเดือน / รายสัปดาห์ / รายวัน
*   **เพิ่มนัดหมาย:** คลิกที่วันที่ → กรอกข้อมูล → บันทึก
*   **สถานะนัดหมาย:** pending / completed / cancelled / no_show
*   **แถบสี:** แสดงสถานะของผู้ป่วยแต่ละนัด

---

## 6. กล่องข้อความ (Inbox)

*   **ข้อความอัตโนมัติ:** ระบบแจ้งเตือนเมื่อสถานะผู้ป่วยเปลี่ยน หรือผลแล็บพร้อม
*   **ข้อความส่วนตัว:** ส่งข้อความหาเพื่อนร่วมงานได้ กดปุ่ม "เขียนข้อความ"
*   **จัดการ:** อ่าน / ตอบกลับ / ลบ / อ่านเป็นกลุ่ม (Bulk)
*   **ป้ายแจ้งเตือน:** ตัวเลขสีแดงที่เมนูแสดงจำนวนข้อความยังไม่อ่าน
*   อัปเดตอัตโนมัติ

---

## 7. โปรไฟล์ (Profile)

*   **ดูข้อมูล:** ชื่อ, อีเมล, ตำแหน่ง, บทบาท, เลขใบประกอบวิชาชีพ
*   **แก้ไขข้อมูล:** แก้ไขชื่อ-นามสกุล, เปลี่ยนรหัสผ่าน, อัปเดต Bio
*   **ตั้งค่า PIN:** สร้างหรือเปลี่ยนรหัส PIN 6 หลักสำหรับ E-Signature

---

## 8. สำหรับผู้ดูแลระบบ (Admin)

### 8.1 แดชบอร์ดผู้ดูแล
*   แสดงสถิติภาพรวม + **ข้อมูลเชิงเทคนิคของระบบ** (Database, Version, Security)

### 8.2 ตารางรายงานสรุป (Reports)
*   กราฟสถิติแยกได้ 5 scope: **รายชั่วโมง / รายวัน / รายสัปดาห์ / รายเดือน / รายปี**
*   สรุปจำนวนยอดตรวจเลือด แยกตามประเภทโรค / ห้องปฏิบัติการ
*   เลือกช่วงเวลา → สั่งพิมพ์รายงานสรุป A4

### 8.3 จัดการผู้ใช้งาน
*   เพิ่มผู้ใช้ / อนุมัติบัญชี / ระงับสิทธิ์ / ลบบัญชี
*   เปลี่ยนบทบาท (Role) ของผู้ใช้
*   **Debug Role Override:** Admin สามารถจำลองบทบาทอื่นเพื่อทดสอบระบบ

### 8.4 หน้า Admin ทั้งหมด

| หน้า | คำอธิบาย |
|------|----------|
| `/admin` | ศูนย์รวมสถิติ + ข้อมูลระบบ |
| `/admin/patients` | จัดการข้อมูลผู้ป่วยทั้งหมด |
| `/admin/doctors` | จัดการรายชื่อแพทย์/บุคลากร |
| `/admin/lab` | จัดการห้องแล็บ |
| `/admin/inbox` | กล่องข้อความทั้งหมดในระบบ |
| `/admin/reports` | รายงานสรุปภาพรวม + กราฟ |

## 9. สิทธิ์การใช้งานตามบทบาท (RBAC)

| ฟีเจอร์ | แพทย์ | พยาบาล | เจ้าหน้าที่แล็บ | ผู้ดูแล |
|---------|:-----:|:------:|:---------:|:-----:|
| ดูข้อมูลผู้ป่วย | ✅ | ✅ | ✅ | ✅ |
| ลงทะเบียนผู้ป่วย | ✅ | ✅ | ❌ | ✅ |
| แก้ไขข้อมูลผู้ป่วย | ✅* | ✅* | ❌ | ✅ |
| อัปเดตสถานะ | ✅ | ✅ | ✅** | ✅ |
| ดู/อัปโหลดผลตรวจ | ✅ (ดู) | ✅ (ดู) | ✅ (ทั้งหมด) | ✅ |
| อนุมัติและพิมพ์ผล | ✅ | ✅ | ✅ | ✅ |
| ส่งข้อความ (Inbox) | ✅ | ✅ | ✅ | ✅ |
| จัดการผู้ใช้ | ❌ | ❌ | ❌ | ✅ |
| ดูรายงาน/กราฟ | ❌ | ❌ | ❌ | ✅ |

\* เฉพาะผู้ป่วยที่ตนเองรับผิดชอบ
\** เฉพาะสถานะที่เกี่ยวข้องกับห้องแล็บเท่านั้น

## 10. การแก้ปัญหาเบื้องต้น (Troubleshooting)
*   **Pop-up Blocker ปฏิเสธการพิมพ์:** เปลี่ยนเป็นกดอนุญาต "Always allow pop-ups" จากเว็บเบราว์เซอร์
*   **กรอก PIN ผิดบ่อยจนบล็อก:** แจ้ง Admin ให้ช่วยรีเซ็ต PIN ใหม่จากหน้าโปรไฟล์
*   **ไม่ได้รับอีเมลรีเซ็ตรหัสผ่าน:** ตรวจสอบโฟลเดอร์ Spam/Junk หรือติดต่อ Admin
