# คู่มือการใช้งานระบบ Bloodlink (User Manual)

เอกสารนี้รวบรวมวิธีการใช้งานระบบสำหรับผู้ใช้งานแต่ละระดับ เพื่อให้เกิดประสิทธิภาพสูงสุดในการจัดการข้อมูลผลเลือด

---

## 1. การเข้าสู่ระบบ และการตั้งค่า PIN
*   เข้าสู่เว็บไซต์ผ่าน URL ของหน่วยงานท่าน
*   กรอก **Email** และ **Password** เพื่อล็อคอิน
*   **สำคัญ (ลายเซ็นอิเล็กทรอนิกส์):** เมื่อเข้าใช้งานครั้งแรก ให้ไปที่เมนู **"โปรไฟล์ (Profile)"** เพื่อทำการตั้งค่ารหัส PIN 6 หลัก หมวดนี้จะใช้สำหรับยืนยันการทำรายการเข้าถึงข้อมูลและการลงนามในใบผลแล็บ

---

## 2. สำหรับเจ้าหน้าที่พยาบาล และแพทย์ (Clinical Staff)

### 2.1 การลงทะเบียนผู้ป่วยใหม่
1.  ไปที่เมนู **"ผู้ป่วย" (Patients)**
2.  กดปุ่ม **"+ ลงทะเบียนผู้ป่วย"**
3.  กรอกข้อมูล: เลข HN, ชื่อ-นามสกุล, อายุ, ประวัติการแพ้ยา และ **ติ๊กเลือกโรคประจำตัว** (สามารถเลือกได้มากกว่า 1 โรค)
4.  กดบันทึก

### 2.2 การบันทึกข้อมูลก่อนส่งตรวจ (Pre-Lab)
1.  ค้นหาผู้ป่วยจากหน้าแดชบอร์ด แล้วกดเข้าสู่ "ประวัติผู้ป่วย"
2.  เลือกแท็บ **"บันทึกก่อนตรวจ (Pre-Lab)"** 
3.  กรอกอัตราการเต้นหัวใจ, ความดัน (BP1, BP2), ส่วนสูง, น้ำหนัก
4.  กดเลือกรายการตรวจเลือดที่แพทย์สั่ง (เช่น FBS, Lipid, CBC, HbA1c) 
5.  กดยืนยัน ระบบจะสร้างคิวใบส่งตรวจไปยังฝั่งห้องปฏิบัติการทันที

### 2.3 การติดตามสถานะคิว
*   สามารถไปที่เมนู **"สถานะผลตรวจเลือด" (Test Status)** เพื่อดูว่าผู้ป่วยอยู่ในขั้นตอนใด (รอแล็บรับ, กำลังปั่นเลือด, หรือผลออกแล้ว)
*   ในหน้านี้ สามารถคลิกดู **"ใบส่งตรวจ (A5)"** และสั่งพิมพ์ย้อนหลังได้ตลอดเวลา

---

## 3. สำหรับเจ้าหน้าที่ห้องปฏิบัติการ (MedTech)

### 3.1 การตรวจสอบคิว และสั่งพิมพ์ใบส่งตรวจแบบปึก (Batch Printing)
1.  ไปที่เมนู **"คิวงานแล็บ" (Lab Queue)** ระบบจะแสดงรายชื่อผู้ป่วยที่พยาบาลเจาะเลือดเสร็จแล้ว
2.  ท่านสามารถตรวจสอบรายละเอียดคิว สั่งพิมพ์ **"ใบปะหน้าสรุป (Summary A4)"** ควบคู่กับ **"ใบรายการตรวจ (Request Sheet A5)"** ได้พร้อมกันหลายๆ คน โดยการติ๊กเลือกผู้ป่วยทั้งหมด แล้วกดปุ่มเครื่องพิมพ์ใหญ่ 
3.  เมื่อพร้อมตรวจ ให้เจ้าหน้าที่กด **"รับคิว"** เพื่อย้ายคิวนั้นเข้ามาอยู่ในหน้าต่าง **"งานของฉัน (My Tasks)"** ทำให้ทำงานไม่ซ้ำซ้อนกับเพื่อนร่วมงาน

### 3.2 การอัปโหลดผลแล็บ (Results Import & OCR)
1.  ในหน้าการตรวจผล ท่านสามารถกรอกผลเลือดด้วยมือ หรือใช้เทคโนโลยีช่วยอ่าน 
2.  กดปุ่ม **"อัปโหลดใบผลตรวจ"** แล้วถ่ายรูปกระดาษแล็บจากเครื่อง Automate ตัวระบบจะใช้เทคโนโลยี **OCR** สกัดตัวเลขต่างๆ มาใส่ในช่องฟอร์มให้อัตโนมัติ!
3.  แก้ไขและตรวจสอบทศนิยมให้ถูกต้องก่อนยืนยัน

### 3.3 การอนุมัติและแจ้งพิมพ์ผลเพื่อเก็บประวัติ (Validation)
1.  เมื่อทำการวิเคราะห์เสร็จสิ้น ให้มาที่เมนู **"ผลตรวจเลือด" (Results)**
2.  เลือกผู้ป่วยที่ต้องการยืนยันผล กดปุ่ม **"พิมพ์เพื่ออนุมัติ"** 
3.  ระบบจะให้ท่านตรวจสอบรูปแบบบัตรผลเลือดอีกครั้ง และบังคับใส่ **รหัส PIN 6 หลัก** ก่อนยืนยันพิมพ์ (เพื่อสร้าง E-Signature ไว้มุมกระดาษ)
4.  เมื่อสั่งพิมพ์เสร็จ การ์ดผู้ป่วย จะหายไปจากกระดานแล็บหน้าจอ เพื่อให้ท่านโฟกัสกับคิวถัดไปได้แบบ Real-time 

---

## 4. สำหรับผู้ดูแลระบบ (Admin Tools)

### 4.1 ตารางรายงานสรุป (Reports Dashboard)
1.  ไปที่เมนู **"รายงาน (Reports)"**
2.  ระบบจะสรุปจำนวนผู้ป่วยที่เข้ารับการตรวจเลือด แยกตามประเภทโรค แยกตามห้องปฏิบัติการ พร้อมกราฟแสดงผลรายวัน 
3.  สามารถกดส่งออก (Print Report) เป็นเอกสารตัวชี้วัดองค์กรได้ตามช่วงเวลาที่ระบุ

### 4.2 การจัดการผู้ใช้งานและสิทธิ์ (RBAC)
1.  เพิ่มผู้ใช้งานใหม่ แบน หรืออนุญาตสิทธิ์การเข้าถึงข้อมูลระบบ ตามการรักษาความปลอดภัย
2.  ระบบมี **Role Guard** และ **Server Actions** ทำให้มั่นใจได้ว่าคำขอในทุกๆ API จะถูกตรวจเช็คสิทธิ์แบบเคร่งครัดเสมอ

---

## 5. การแก้ปัญหาเบื้องต้น (Troubleshooting)
*   **Pop-up Blocker ปฏิเสธการพิมพ์ผลง:** ระบบรุ่นใหม่ใช้ Snapshot Database ทำให้ปัญหานี้น้อยลง แต่ถ้าเจอปัญหา ให้เปลี่ยนเป็นกดอนุญาต "Always allow pop-ups" จากเว็บบราวเซอร์
*   **กรอกคะแนน PIN ผิดบ่อยจนบล็อก:** แจ้งแอดมินให้ช่วยสิทธิเคลียร์ หรือรีเซ็ต PIN ใหม่จากหน้าโปรไฟล์
