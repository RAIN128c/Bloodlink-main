# รายละเอียดโครงงานและวิธีการดำเนินงาน (Project Details & Methodology)

## ส่วนที่ 1: ทฤษฎีและหลักการที่เกี่ยวข้อง (Theory)

### 1.1 บริบทของระบบ (Context)
*   **ระบบส่งต่อผู้ป่วย:** ช่วยลดความล่าช้าในการส่งข้อมูลระหว่าง รพ.สต. และโรงพยาบาลแม่ข่าย
*   **ระบบห้องปฏิบัติการ (LIS):** การจัดการข้อมูลผลเลือด (CBC) ให้เป็นดิจิทัล ลดความผิดพลาดจากการเขียนมือ

### 1.2 เทคโนโลยีที่เลือกใช้ (Technology Stack)
*   **Web Application:** เลือกใช้สถาปัตยกรรมแบบ Client-Server เพื่อให้เข้าถึงได้จากทุกอุปกรณ์
*   **Next.js:** ใช้สำหรับการทำ Server-Side Rendering (SSR) เพื่อประสิทธิภาพและความเร็ว
*   **Tailwind CSS:** เพื่อการออกแบบที่รองรับหน้าจอทุกขนาด (Responsive) อย่างรวดเร็ว
*   **Supabase:** ใช้เป็นฐานข้อมูล PostgreSQL ที่รองรับการแจ้งเตือนแบบ Real-time (WebSockets)

### 1.3 ความปลอดภัย (Security)
*   **PDPA:** การออกแบบระบบคำนึงถึงการคุ้มครองข้อมูลส่วนบุคคล เก็บข้อมูลเท่าที่จำเป็น
*   **Encryption:** ข้อมูลสำคัญจะถูกเข้ารหัสก่อนจัดเก็บ

## ส่วนที่ 2: วิธีการดำเนินงาน (Development Methodology)

การพัฒนาโครงงานนี้ใช้วงจรการพัฒนาระบบ (SDLC) ดังนี้:

### ขั้นตอนที่ 1: การรวบรวมความต้องการ (Requirement Gathering)
*   ศึกษาปัญหาความล่าช้าในการส่งผลแล็บ
*   สอบถามความต้องการจากผู้ใช้งานจริง (แพทย์, พยาบาล)

### ขั้นตอนที่ 2: การออกแบบ (System Design)
*   ออกแบบ **ER Diagram** เพื่อวางแผนโครงสร้างฐานข้อมูล (ดูไฟล์ SYSTEM_ANALYSIS_TH.md ประกอบ)
*   ออกแบบหน้าจอ (UI) เน้นความสะอาดตาและใช้งานง่าย

### ขั้นตอนที่ 3: การพัฒนา (Implementation)
*   **Environment:** พัฒนาบน VS Code
*   **Coding:** เขียนโค้ดด้วยภาษา TypeScript
*   **Database:** เชื่อมต่อกับ Supabase

### ขั้นตอนที่ 4: การทดสอบ (Testing)
*   **Unit Test:** ทดสอบฟังก์ชันย่อย (เช่น การคำนวณค่าผิดปกติ)
*   **UAT (User Acceptance Test):** ให้ผู้ใช้ทดลองใช้งานจริงและรับฟังข้อเสนอแนะ

### ขั้นตอนที่ 5: การติดตั้งและใช้งานจริง (Deployment)
*   นำระบบขึ้น Server จริงผ่าน Vercel
*   ติดตามผลการใช้งานและปรับปรุงแก้ไข (Maintenance)
